# 常见问题

## 拉取请求 (PRs)

### 我如何允许维护者编辑我的 PR？

当你提交一个拉取请求时，可能会有额外的代码变化，
需要在合并之前进行处理。通常情况下，更有效率的是
让维护者自己进行这些更改，而不是让你
在代码审查中去更改。

默认情况下，大多数拉取请求会有一个
`✅ 维护者被允许编辑此拉取请求。`
徽章在右侧边栏中。

如果您没有看到此徽章，您可能已为您正在进行的分叉关闭了此设置。
请参阅[此Github文档页面](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork)以获取更多信息。
有关更多信息。

值得注意的是，Github不允许在**组织**中的分叉上启用此设置（[问题](https://github.com/orgs/community/discussions/5634)）。
如果您在一个组织中工作，我们建议您从个人分叉提交您的PR，
以便启用此设置。

### 为什么我的PR没有被审查？

### 为什么我的PR被关闭了？

请参考我们的[审查流程](/docs/contributing/review_process/)。
